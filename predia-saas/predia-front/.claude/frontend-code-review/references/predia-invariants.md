# Predia Invariants

Stable predia-specific runtime rules. Apply in addition to generic review packs.

Only keep a rule here when all are true:
- Stable predia runtime invariant.
- Generic React, TypeScript, a11y, query, or performance rules would not catch it.
- Failure mode is concrete enough to produce a file-line finding.
- Rule likely remains valid across normal feature work.

## Auth Token Handling

Flag:

- Direct `localStorage` / `sessionStorage` access for tokens outside `src/shared/lib/tokens.ts`.
- Components reading auth state directly from storage instead of via the auth service or query hooks.
- Token refresh logic duplicated outside the API client layer.

All token reads/writes go through `src/shared/lib/tokens.ts`. API client in `src/shared/lib/api.ts` handles auth headers and refresh.

## TanStack Router — Route Tree

Flag:

- Manual edits to `src/routeTree.gen.ts`. This file is auto-generated by the Vite plugin on route file changes.
- Route-level auth guards implemented inside page components instead of in the route loader or parent layout route.
- Navigation via `window.location` or raw `history` instead of TanStack Router APIs.

New routes require only a new file under `src/routes/`. The plugin regenerates `routeTree.gen.ts` automatically.

## API Client Contract

Flag:

- Direct `fetch` or `axios` calls bypassing the `createApiClient` wrapper from `@predia/api-types`.
- Re-declared API response shapes in components instead of importing generated types from `@predia/api-types`.
- Query keys that omit tenant/user identity when the query varies by tenant.
- Hardcoded `VITE_API_URL` base URL strings; must use `import.meta.env.VITE_API_URL`.

Use `createApiClient(import.meta.env.VITE_API_URL)` at `src/shared/lib/api.ts`. Consume generated types, not manual DTOs.

## Feature Module Boundaries

Flag:

- Cross-feature imports that bypass the feature's public `index.ts` or `components/` barrel.
- Shared utilities placed in `src/app/[feature]/` instead of `src/shared/`.
- Feature-specific types imported directly by other feature modules.
- Page components placed outside `src/app/[feature]/components/[Feature]Page.tsx` convention.

Each feature is self-contained. Cross-feature shared code belongs in `src/shared/`.

## AppLayout Skipping

Flag:

- Routes that opt out of `AppLayout` without an explicit pathname check in `__root.tsx`.
- Login/public routes that accidentally render the sidebar/navbar.

Currently `__root.tsx` skips `AppLayout` for `/login` via pathname check. Any new public routes must follow the same pattern.
